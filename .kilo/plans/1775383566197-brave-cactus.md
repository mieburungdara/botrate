# Rencana Pembuatan Bot Telegram Album Media

## Ringkasan
Bot Telegram untuk menerima album foto/video, moderasi, dan distribusi terkontrol menggunakan Node.js dan MySQL InnoDB.

## Fitur Utama
1. Menerima SEMUA jenis media: foto tunggal, video tunggal, dan album media grup (campuran foto/video)
2. Menyimpan informasi pengguna dan semua pesan media
3. Meneruskan ke channel moderator dengan inline keyboard (Setuju/Batal)
4. Setelah disetujui, buat link unik dan kirim ke channel publik
5. Pengguna yang mengklik link akan menerima media asli dengan caption
6. Fitur rating album 1-5 bintang setelah media diterima
7. Mini Web App untuk melihat daftar album dan statistik pribadi

## Arsitektur Teknis

### Stack Teknologi
- **Runtime**: Node.js (LTS versi terbaru)
- **Library Telegram**: `telegraf` (v4.x) - framework bot modern
- **Database**: MySQL 8.0+ dengan engine InnoDB
- **Driver Database**: `mysql2` dengan prepared statements
- **Environment**: `dotenv` untuk variabel lingkungan
- **Metode Koneksi**: Webhook (membutuhkan domain dengan HTTPS)

### Struktur Database
Tabel yang dibutuhkan:

1. `users`
   - `user_id` (BIGINT, PRIMARY KEY) - Telegram user ID
   - `username` (VARCHAR(255))
   - `first_name` (VARCHAR(255))
   - `last_name` (VARCHAR(255))
   - `is_admin` (BOOLEAN DEFAULT FALSE)
   - `last_active` (DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP)
   - `album_count` (INT DEFAULT 0) - jumlah album yang dikirim
   - `download_count` (INT DEFAULT 0) - jumlah album yang diunduh
   - `created_at` (DATETIME DEFAULT CURRENT_TIMESTAMP)
   - `updated_at` (DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP)

2. `albums`
   - `id` (INT, PK, AUTO_INCREMENT)
   - `user_id` (BIGINT, FOREIGN KEY REFERENCES users(user_id))
   - `message_ids` (JSON) - array ID pesan album
   - `chat_id` (BIGINT) - chat ID pengguna asli
   - `caption` (TEXT) - caption album
   - `unique_token` (VARCHAR(32), UNIQUE) - token untuk link
   - `status` (ENUM('pending', 'approved', 'rejected'))
   - `moderator_message_id` (BIGINT) - ID pesan di channel moderator
   - `download_count` (INT DEFAULT 0) - jumlah unduhan
   - `rating_count` (INT DEFAULT 0) - jumlah rating yang diterima
   - `rating_total` (INT DEFAULT 0) - total nilai rating
   - `rating_avg` (DECIMAL(3,2) DEFAULT 0.00) - rata-rata rating
   - `created_at` (DATETIME DEFAULT CURRENT_TIMESTAMP)
   - `approved_at` (DATETIME)
   - `rejected_at` (DATETIME)

3. `ratings`
   - `id` (INT, PK, AUTO_INCREMENT)
   - `album_id` (INT, FOREIGN KEY REFERENCES albums(id) ON DELETE CASCADE)
   - `user_id` (BIGINT, FOREIGN KEY REFERENCES users(user_id))
   - `rating` (TINYINT NOT NULL) - 1-5 bintang
   - `comment` (TEXT NULL) - komentar opsional
   - `created_at` (DATETIME DEFAULT CURRENT_TIMESTAMP)
   - UNIQUE KEY `unique_rating` (`album_id`, `user_id`) - Satu user hanya bisa rate satu album sekali

4. `blacklist`
   - `id` (INT, PK, AUTO_INCREMENT)
   - `user_id` (BIGINT, UNIQUE, FOREIGN KEY REFERENCES users(user_id))
   - `reason` (TEXT)
   - `banned_by` (BIGINT, FOREIGN KEY REFERENCES users(user_id)) - admin ID yang memblokir
   - `created_at` (DATETIME DEFAULT CURRENT_TIMESTAMP)

4. `activity_logs`
   - `id` (INT, PK, AUTO_INCREMENT)
   - `action` (VARCHAR(50)) - 'approve', 'reject', 'ban', 'unban', 'download', 'send'
   - `user_id` (BIGINT, FOREIGN KEY REFERENCES users(user_id)) - user yang melakukan aksi
   - `target_id` (BIGINT) - ID album atau user yang diaksi
   - `details` (TEXT)
   - `created_at` (DATETIME DEFAULT CURRENT_TIMESTAMP)

5. `settings`
   - `key` (VARCHAR(50), PK)
   - `value` (TEXT)

### Alur Kerja

1. **Auto-Register User**
   - Setiap interaksi user dengan bot akan otomatis register/update data user di tabel `users`
   - Update `last_active` setiap kali user berinteraksi
   - Cek status blacklist sebelum memproses permintaan apapun

2. **Penerimaan Album**
   - Bot mendeteksi pesan media grup (album)
   - Mengumpulkan semua pesan dalam album (karena Telegram mengirim album sebagai pesan terpisah)
   - Menyimpan semua ID pesan dan relasi dengan user_id dari tabel `users`
   - Increment `album_count` user di tabel `users`
   - Meneruskan ke channel moderator dengan inline keyboard

2. **Moderasi**
   - Admin menekan tombol "Setuju" atau "Batal"
   - Update status album di database
   - Jika disetujui: buat token unik, buat link `t.me/botusername?start=token`
   - Kirim pesan ke channel B dengan link tersebut
   - Edit pesan moderator untuk menampilkan status

3. **Distribusi**
   - Pengguna mengklik link dan memulai bot dengan parameter token
   - Bot memverifikasi token valid dan album sudah disetujui
   - Menggunakan `copyMessages` untuk mengirim seluruh album ke pengguna
   - Menyertakan caption asli dari pengirim
   - **SETELAH ALBUM TERKIRIM**: Menampilkan tombol rating 1-5 bintang untuk pengguna

4. **Rating Album**
   - Pengguna menekan tombol rating (⭐ 1 - ⭐⭐⭐⭐⭐ 5)
   - Rating disimpan di tabel `ratings` (satu user hanya bisa rate satu album sekali)
   - Update statistik rating album (`rating_count`, `rating_total`, `rating_avg`)
   - Konfirmasi terima kasih setelah rating sukses
   - Rating rata-rata ditampilkan di pesan channel publik dan di pesan moderator

5. **Mini Web App Telegram**
   - Akses via tombol menu bot atau inline keyboard
   - Autentikasi otomatis menggunakan data init Telegram Web App
   - Tampilan daftar album yang dikirim pengguna beserta status dan rating
   - Tampilan statistik pribadi (jumlah album dikirim, jumlah album diunduh, rata-rata rating)
   - Tampilan riwayat download
   - Admin panel untuk melihat statistik global dan manajemen album
   - Responsif dan dioptimalkan untuk tampilan mobile Telegram

- Validasi semua input pengguna dan parameter callback
- Batasi ukuran album (maks 10 media per album sesuai batas Telegram)
- Handle kasus album yang tidak lengkap (pesan terpotong)
- Rate limiting untuk mencegah spam pengiriman album
- Hanya admin yang dapat menekan tombol moderasi
- Token unik menggunakan crypto secure random
- Semua query database menggunakan prepared statements
- Sanitasi output untuk mencegah injection di pesan Telegram

## Edge Cases yang Perlu Ditangani
1. Album dengan campuran foto dan video
2. Pengguna mengirim album parsial (kurang dari 10)
3. Pesan moderator dihapus sebelum di moderasi
4. User diblokir mid-pengiriman album
5. Token kedaluarsa (jika dibutuhkan)
6. Batas rate Telegram API tercapai
7. Media sudah dihapus dari server Telegram
8. User non-admin menekan tombol moderasi
9. Album tidak lengkap karena timeout pengumpulan pesan
10. Caption melebihi batas maksimal Telegram (1024 karakter)
11. User mencoba memberikan rating lebih dari sekali
12. User mencoba rating album yang belum mereka download
13. Rating dengan nilai tidak valid (kurang dari 1 atau lebih dari 5)
14. Media tunggal yang dikirim tanpa album
15. Campuran foto dan video dalam satu album

## Daftar Bug yang Ditemukan dan Perlu Diperbaiki

### Bug Kritis
1. **`copyMessages` parameter salah**
   - Lokasi: `src/handlers/album.js:76`
   - Masalah: Parameter `copyMessages` urutannya salah. Parameter pertama seharusnya `fromChatId`, kedua `toChatId`, bukan sebaliknya.
   - Dampak: Album tidak akan pernah terkirim ke channel moderator.

2. **`copyMessages` tidak mengembalikan message_id**
   - Lokasi: `src/handlers/album.js:85`
   - Masalah: `copyMessages` mengembalikan array of Message, bukan object tunggal. Akses `moderatorMessage.message_id` akan error.

3. **User belum dicek blacklist sebelum memproses**
   - Lokasi: Semua handler
   - Masalah: Middleware blacklist belum diimplementasikan dan tidak terpasang di bot. User yang diblokir tetap dapat mengirim media.

4. **Rating dihitung salah**
   - Lokasi: `src/handlers/distribution.js:91`
   - Masalah: Update rating menggunakan `rating_count + 1` sebelum di-increment. Nilai akan selalu salah satu angka.
   - Perbaikan: Gunakan `rating_count + 1` untuk pembagi, atau increment dulu sebelum update.

### Bug Medium
5. **Album cache tidak dibersihkan jika user diblokir**
   - Lokasi: `src/handlers/album.js`
   - Masalah: Jika user diblokir saat sedang mengirim album, album tetap akan diproses setelah timeout.

6. **Tidak ada pengecekan maksimal media per album**
   - Lokasi: `src/handlers/album.js`
   - Masalah: User dapat mengirim album dengan lebih dari 10 media yang akan menyebabkan error Telegram API.

7. **Tidak ada error handling di Web App API**
   - Lokasi: Semua endpoint API
   - Masalah: Jika query database gagal, API akan crash dan tidak mengembalikan response.

8. **`message_id` media tunggal tidak disimpan dengan benar**
   - Lokasi: `src/handlers/album.js:133`
   - Masalah: `copyMessage` mengembalikan objek pesan, namun `moderatorMessage.message_id` diakses sebelum divalidasi.

### Bug Minor
9. **Web App tidak menangani kasus rating 0**
   - Lokasi: `webapp/app.js`
   - Masalah: Jika album belum memiliki rating, nilai 0 akan ditampilkan tanpa penanganan khusus.

10. **Pesan channel publik tidak diupdate ketika rating berubah**
    - Lokasi: `src/handlers/moderation.js`
    - Masalah: Pesan di channel publik hanya dikirim sekali dengan rating 0. Tidak pernah diupdate meskipun sudah banyak yang memberikan rating.

11. **Tidak ada pengecekan duplikat album**
    - Masalah: User dapat mengirim album yang sama berkali-kali tanpa batas.

12. **Timeout album tidak dibersihkan ketika proses selesai**
    - Masalah: Memory leak karena timeout tidak dibersihkan secara konsisten.

13. **Admin user pertama tidak diinsert otomatis**
    - Masalah: Admin pertama perlu diinsert manual ke database, tidak ada mekanisme otomatis.

### Masalah Desain
14. **Tidak ada index di field `unique_token` selain UNIQUE**
    - Masalah: Query pencarian token akan lambat ketika data banyak.

15. **Tidak ada batas ukuran caption**
    - Masalah: Caption yang lebih dari 1024 karakter akan menyebabkan error Telegram API.

16. **Tidak ada retry mechanism untuk API Telegram**
    - Masalah: Jika API Telegram mengalami gangguan sekali, proses akan gagal total.

17. **Rating dapat diberikan tanpa mendownload album**
    - Masalah: User dapat memanggil callback `rate_*` secara langsung tanpa pernah mendownload album terlebih dahulu.

## Hal yang Perlu Ditambahkan (Opsional)

### 1. Keamanan Webhook
- **Webhook Secret Token**: Verifikasi header `X-Telegram-Bot-Api-Secret-Token` di setiap request masuk untuk mencegah request palsu
- **Allowed Updates**: Batasi jenis update yang diterima bot hanya yang dibutuhkan (`message`, `callback_query`, `edited_message`)

### 2. Mekanisme Pengumpulan Album
- **Timeout 5 detik**: Telegram mengirim album sebagai pesan terpisah. Perlu mekanisme untuk menunggu semua pesan dengan `media_group_id` yang sama terkumpul sebelum diproses
- **Cache sementara**: Simpan pesan album yang belum lengkap di memory cache dengan TTL

### 3. Validasi Akses Moderasi
- **Pengecekan Admin**: Hanya user ID yang terdaftar sebagai admin di tabel `users` yang dapat menekan tombol Setuju/Batal
- **Callback query validation**: Verifikasi `from.id` pada setiap callback query sebelum memproses

### 4. Notifikasi Pengirim
- Kirim pesan konfirmasi ke pengirim ketika album disetujui/ditolak
- Sertakan alasan penolakan jika admin memberikan catatan

### 5. Rate Limiting
- Batasi jumlah album yang dapat dikirim per user (misal 5 album per jam)
- Batasi jumlah download per user per jam untuk mencegah abuse

### 6. Error Handling
- Retry mechanism untuk API Telegram dengan exponential backoff
- Penanganan error ketika `copyMessages` gagal di tengah
- Alert admin jika terjadi error fatal

### 7. Fitur Tambahan Disetujui
- **Alasan penolakan dengan tombol preset**: Saat admin menekan tombol "Batal", akan muncul pilihan tombol alasan penolakan standar:
  - Konten tidak sesuai
  - Kualitas rendah
  - Duplikat
  - Melanggar aturan
  - Lainnya
- Token download PERMANEN (tidak kadaluarsa)
- Pemberitahuan ke pengirim album ketika disetujui/ditolak beserta alasan penolakan

### Helper Web App (src/helpers/webapp.js)
```javascript
const crypto = require('crypto');

function verifyWebAppInitData(initData) {
    const params = new URLSearchParams(initData);
    const hash = params.get('hash');
    params.delete('hash');

    const dataCheckString = Array.from(params.entries())
        .sort(([a], [b]) => a.localeCompare(b))
        .map(([key, value]) => `${key}=${value}`)
        .join('\n');

    const secretKey = crypto.createHmac('sha256', 'WebAppData')
        .update(process.env.BOT_TOKEN)
        .digest();

    const calculatedHash = crypto.createHmac('sha256', secretKey)
        .update(dataCheckString)
        .digest('hex');

    return calculatedHash === hash;
}

function parseInitData(initData) {
    const params = new URLSearchParams(initData);
    const user = JSON.parse(params.get('user') || '{}');
    return {
        user,
```

```javascript
// Cache untuk menyimpan album yang sedang dikumpulkan
const albumCache = new Map();
const ALBUM_TIMEOUT = 5000; // 5 detik

async function handleAlbum(ctx) {
    const mediaGroupId = ctx.update.message.media_group_id;
    const userId = ctx.from.id;
    const chatId = ctx.chat.id;
    const messageId = ctx.message.message_id;
    const caption = ctx.message.caption || '';

    if (!albumCache.has(mediaGroupId)) {
        albumCache.set(mediaGroupId, {
            user_id: userId,
            chat_id: chatId,
            message_ids: [],
            caption: caption,
            timeout: setTimeout(async () => {
                await processAlbum(mediaGroupId, ctx);
            }, ALBUM_TIMEOUT)
        });
    }

    const album = albumCache.get(mediaGroupId);
    album.message_ids.push(messageId);
    
    if (caption && !album.caption) {
        album.caption = caption;
    }

    clearTimeout(album.timeout);
    album.timeout = setTimeout(async () => {
        await processAlbum(mediaGroupId, ctx);
    }, ALBUM_TIMEOUT);
}

async function processAlbum(mediaGroupId, ctx) {
    const album = albumCache.get(mediaGroupId);
    if (!album) return;
    
    albumCache.delete(mediaGroupId);
    clearTimeout(album.timeout);

    try {
        const token = generateToken();
        
        // Simpan album ke database

            INSERT INTO albums (user_id, message_ids, chat_id, caption, unique_token)
            VALUES (?, ?, ?, ?, ?)

        await db.execute(
            'UPDATE users SET album_count = album_count + 1 WHERE user_id = ?',
            [album.user_id]
        );

        // Dapatkan info user
        const [userRows] = await db.execute(
            'SELECT username, first_name FROM users WHERE user_id = ?',
            [album.user_id]
        );
        const user = userRows[0];

                reply_markup: keyboard.reply_markup
            }
        );

        // Simpan message_id moderator
```

```javascript
                ]

async function handleRating(ctx) {
    const albumId = ctx.match[1];
    const rating = parseInt(ctx.match[2]);

    if (rating < 1 || rating > 5) {
        return ctx.answerCbQuery('Rating tidak valid', true);
    }

    // Cek apakah user sudah pernah rate
    const [existingRating] = await db.execute(
        'SELECT id FROM ratings WHERE album_id = ? AND user_id = ?',
        [albumId, ctx.from.id]
    );

    if (existingRating.length > 0) {
        return ctx.answerCbQuery('Anda sudah memberikan rating untuk album ini', true);
    }

    try {
        // Simpan rating
        await db.execute(`
            INSERT INTO ratings (album_id, user_id, rating)
            VALUES (?, ?, ?)
        `, [albumId, ctx.from.id, rating]);

        // Update statistik album
        await db.execute(`
            UPDATE albums 
            SET 
                rating_count = rating_count + 1,
                rating_total = rating_total + ?,
                rating_avg = ROUND((rating_total + ?) / (rating_count + 1), 2)
            WHERE id = ?
        `, [rating, rating, albumId]);

        // Edit pesan rating
        await ctx.editMessageText(`Terima kasih! Anda memberikan rating ${'⭐'.repeat(rating)} untuk album ini.`);
        await ctx.answerCbQuery('Rating berhasil disimpan!');

    } catch (error) {
        console.error('Rating error:', error);
        await ctx.answerCbQuery('Gagal menyimpan rating', true);
    }
}
```
